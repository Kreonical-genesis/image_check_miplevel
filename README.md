# ResourcePack Mipmap Checker

Web-инструмент для анализа текстур ресурспаков Minecraft на поддержку mipmapping.

![Preview](https://img.shields.io/badge/HTML5-CSS3-green) ![JS](https://img.shields.io/badge/JavaScript-ES6+-yellow) ![No Backend](https://img.shields.io/badge/Backend-None-blue)

## Что это?

Инструмент проверяет текстуры в ZIP/JAR архивах на корректность размеров для mipmapping в Minecraft. Одна проблемная текстура может снизить качество mipmap для **всех** текстур сразу.

## Использование

1. Откройте `index.html` в браузере
2. Перетащите ZIP или JAR файл в область загрузки
3. Получите результат анализа

## Интерфейс

- **Статистика** — количество текстур по категориям
- **Критичные (< 2 уровней)** — текстуры с недостаточным mipmapping
- **Ограниченные (2-3 уровня)** — частичная поддержка
- **Полная поддержка (4+)** — оптимальные размеры

## Что такое Mipmap?

**Mipmap Levels** — это настройка графики в Minecraft, которая делает текстуры вдалеке более гладкими и менее "рябящими", одновременно улучшая производительность.

Простыми словами, это создание уменьшенных копий текстур для объектов вдали, чтобы вместо множества мелких пикселей игра использовала заранее сглаженное изображение.

### Как это работает?

Когда вы смотрите на объект вблизи, игра использует текстуру с максимальным разрешением. Но когда объект находится далеко, он занимает всего несколько пикселей на экране. Если пытаться "втиснуть" детальную текстуру, возникнет **алиасинг** (лесенка) и мерцание.

Mipmapping решает эту проблему заранее:

1. Для каждой текстуры создается цепочка (пирамида) копий, каждая в два раза меньше предыдущей
2. Когда игра рендерит удаленный объект, она выбирает копию подходящего размера
3. Дальние объекты выглядят сглаженными, а видеокарта тратит меньше ресурсов

### Настройка в игре

В настройках графики Minecraft (`Options → Video Settings`) параметр **Mipmap Levels** принимает значения от 0 до 4:

- **0 (OFF)** — Мип-маппинг отключен. Текстуры будут резкими, но могут "рябить"
- **1-4** — Включение мип-маппинга. Чем выше значение, тем для более дальних объектов используются уменьшенные копии

## Требования к текстурам

**Mipmap levels** = сколько раз размер делится на 2 без остатка.

| Размер | Mip Levels | Статус             |
|--------|------------|--------------------|
| 64x64  | 6          | ✓ Полная поддержка |
| 32x32  | 5          | ✓ Полная поддержка |
| 16x16  | 4          | ✓ Полная поддержка |
| 8x8    | 3          | ⚠ Ограниченная     |
| 4x4    | 2          | ⚠ Ограниченная     |
| 2x2    | 1          | ✗ Критично         |
| 20x20  | 1          | ✗ Критично (NPOT)  |

### NPOT (Non-Power-of-Two)

Текстуры с размерами не кратными степени двойки (20x20, 48x48) не могут иметь полный mipmapping, так как цепочка делений прерывается.

## ⚠️ Важные нюансы

### Глобальное влияние

Если **любая одна** текстура не может создать полную цепочку мип-уровней, уровень падает для **всех** текстур. В логах появится: `limits mip level from X to Y`

### Только для блоков/предметов

Mipmap применяется **только к атласу блоков/предметов** (terrain.png, items.png). GUI текстуры (меню, инвентарь) используют отдельный атлас и не зависят от этой проверки.

### Hi-Res паки

Для ресурс-паков с высоким разрешением (256x256, 512x512) стандартных 4 уровней может не хватать. Существуют моды:

- **Better Mipmaps** — добавляет уровни mipmap
- **TexTweaks** — расширенные настройки текстур

### Кастомные модели блоков

Если вы используете только часть текстуры 16x16 (например, область 9x9), остальная часть файла все равно участвует в создании мип-уровней. Контрастные цвета в неиспользуемой области могут создавать артефакты на дальних расстояниях. Решение — заливать неиспользуемую часть цветом, близким к основному.
